Migrate All Third-Party Integrations to Environment Variables
This plan details the process for completely removing all third-party integration settings from the Admin UI and database. All third-party services (Map, Firebase, SMS, Payment Gateway, and Mail) will be configured exclusively via environment variables (.env).

Proposed Changes

[MODIFY] Backend/src/config/env.js
Export all the aforementioned environment variables so they can be securely accessed throughout the backend codebase.
Backend Components
[DELETE] Backend/src/modules/taxi/admin/models/AdminThirdPartySetting.js
Delete the model entirely, as we are no longer storing any third-party settings in the database.
[DELETE] Backend/src/modules/taxi/admin/data/defaultThirdPartySettings.js
Delete this file since default settings are now managed via environment variables.
[MODIFY] Backend/src/modules/taxi/admin/services/adminService.js
Remove all getters and setters for third-party settings:
getPaymentSettings / updatePaymentSettings
getSMSSettings / updateSMSSettings
getFirebaseSettings / updateFirebaseSettings
getMapSettings / updateMapSettings
getMailSettings / updateMailSettings
Remove references to AdminThirdPartySetting and ensureThirdPartySettings().
[MODIFY] Backend/src/modules/taxi/admin/controllers/adminController.js
Delete corresponding controller methods for fetching and updating integration settings.
[MODIFY] Backend/src/modules/taxi/admin/routes/adminRoutes.js
Delete all GET and POST/PATCH routes pointing to /admin/integration-settings/*.
[MODIFY] Backend/src/modules/taxi/services/paymentGatewayService.js
Hardcode the active gateway logic to return Razorpay by default.
Refactor resolveConfiguredGatewayCredentials to pull keyId and keySecret directly from the env config objects (env.razorpayTestApiKey, env.razorpayLiveApiKey, etc.) based on env.razorpayEnvironment.
[MODIFY] Backend/src/modules/taxi/services/smsService.js
Remove any fallback logic that checks the database for SMS India Hub credentials or useDefaultOtp flags. Make it rely solely on the .env configuration.
Frontend Components
[MODIFY] Frontend/src/modules/taxi/admin/services/adminService.js
Remove integration API endpoints (getPaymentSettings, getMailSettings, etc.) from the Axios service definition.
[MODIFY] Frontend/src/modules/taxi/admin/components/AdminLayout.jsx
Remove the navigation links in the Admin sidebar for:
Map Settings
Firebase Settings

SMS Gateways
Mail Settings

Frontend/src/modules/taxi/admin/pages/settings/FirebaseSettings.jsx
Frontend/src/modules/taxi/admin/pages/settings/PaymentGateways.jsx

Frontend/src/modules/taxi/admin/pages/settings/MailSettings.jsx
[MODIFY] Frontend/src/App.jsx (or respective routing configs)
Remove the routes for the deleted settings pages.
Verification Plan
Automated Verification
Run server initialization to ensure no crashes occur due to missing AdminThirdPartySetting imports.
Manual Verification
Admin Panel Checks: Navigate to the admin settings and confirm that Map, Firebase, Payment, SMS, and Mail settings are no longer available in the UI.
Payment Flow: Initiate a Razorpay payment test and confirm the payload generates using credentials stored in .env.
SMS OTP: Test the user onboarding flow to trigger an OTP via SMS India Hub to confirm it authenticates using .env variables.
Google Maps: Verify that map renderings on the live tracking and geofencing pages load successfully using VITE_GOOGLE_MAPS_API_KEY.
